# 新方向启动调研与假设审计 SKILL

> 每次启动新方向前，项目经理解锁本 SKILL，分派给各角色。
> 各角色独立调研，输出真实数据（不推测）。
> 项目经理汇总核验，发现假设依赖就退回重做。

## 项目经理 — 启动指令

```
新方向：[方向名称]

按以下流程走，各角色输出自己的调研结论，但每一步都做一次"假设审计"。

假设审计标准：
  这个结论依赖的是什么假设？
  这个假设有没有被真实数据验证过？
  一个反例是否就能推翻这个方向？
  
如果核心假设没有真实数据支撑，项目经理退回要求补充。
如果核心假设被反例推翻，直接停。
```

## 市场角色调研指令

```
收入规模假设：
  这个方向说"市场有多大"——引用来源是什么？
  来源里说的是"整个行业"还是"这个细分"？
  有没有真实的销售额、用户数、价格数据（不是估算）？
  → 如果没有真实交易数据，标记为"未验证"。

获客路径假设：
  这个方向的第一个用户怎么找到我们？
  我们自己（没有预算、没有团队）能做这个获客动作吗？
  获客成本有没有真实的同行数据（不是"我觉得"）？
  → 如果第一个用户的获客路径不成立，标记为"方向不可行"。

价格验证假设：
  目标用户实际为同类产品付过多少钱？
  这个价格是参考了什么真实定价（不是推测用户愿意付多少）？
  → 如果用户从未为同类产品付过费，标记为"高风险"。
```

## 后端/数据角色调研指令

```
数据可用性假设：
  这个方向依赖的数据源，我们能不能真实获取？
  获取门槛是什么（API Key、付费、爬虫、人工录入）？
  获取到的数据质量如何（完整度、准确度、时效性）？
  → 如果核心数据拿不到或质量不可用，标记为"数据不可行"。

交付物假设：
  这个方向最终交付给用户的是什么？
  这个交付物的技术复杂度在我们能力范围内吗？
  需要什么外部依赖（云服务、第三方 API、合规审批）？
  → 如果交付物有外部依赖不在控制范围内，标记为"依赖风险"。
```

## 产品角色调研指令

```
用户真实反馈（不是推测）：
  有没有真实用户说过"我需要这个"或者"我愿意为这个付钱"？
  这个需求是用户主动表达的，还是我们从行为推测的？
  → 如果没有真实用户说过"要"或"付"，标记为"推测需求"。

竞品真实数据：
  这个方向的竞品有多少真实用户（不是"可能有很多"）？
  竞品的价格是多少（真实的标价，不是估值）？
  竞品活了多少年、融资多少、死了多少？
  → 如果竞品数量为 0 或竞品全死了，标记为"需求不存在"。

替代品验证：
  用户现在用什么解决问题？（纸笔、Excel、别的软件、还是不管了？）
  替代品的成本是多少（时间/金钱）？
  用户对替代品的满意度如何？
  → 如果替代品已经免费且用户满意，标记为"价值不足"。
```

## 项目经理 — 汇总与核验

```
各角色输出后，项目经理做假设审计汇总：

方向：[方向名称]

核心假设清单：
  假设1：[内容] → 验证状态：[已验证 / 未验证 / 被推翻]
  假设2：[内容] → 验证状态：[已验证 / 未验证 / 被推翻]
  假设N：[内容] → 验证状态：[已验证 / 未验证 / 被推翻]

如果任意核心假设为"未验证"或"被推翻"：
  → 退回要求补充数据，或直接停。

决策建议：
  [推进 / 补充数据 / 停]
```
