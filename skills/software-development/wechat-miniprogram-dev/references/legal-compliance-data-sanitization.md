# 开店教练 · 数据合规与品牌名脱敏规范

## 核心原则

- **不违法**（最高优先级规则）
- 法务底线：不展示可识别到特定个人或小微商家的数据
- 竞品数据中保留大品牌（千店+连锁）名称，脱敏中小品牌

## 品牌名脱敏标准

### ✅ 可以保留的（全国/全球知名品牌）

这些品牌已经是公众常识，在竞品分析表中使用属合理商业分析：

| 品类 | 安全品牌 | 原因 |
|------|---------|------|
| 茶饮 | 蜜雪冰城、喜茶、瑞幸、星巴克 | 全国3000+门店，公开上市公司或公众品牌 |
| 火锅 | 海底捞、小龙坎 | 全国1000+门店，上市/准上市公司 |
| 便利店 | 美宜佳、罗森、7-Eleven、全家 | 全国2000+门店，国际品牌 |
| 服装 | 优衣库、ZARA | 全球品牌，公开财报 |
| 咖啡 | 库迪咖啡 | 全国6000+门店 |
| 烘焙 | 好利来、奈雪的茶 | 全国800+门店 |
| 面食 | 和府捞面 | 全国400+门店 |

### 判断标准

```
门店数 < 1000 且 非上市公众公司 → 必须脱敏
门店数 ≥ 1000 或 全球知名品牌   → 可保留，但remark仅限事实描述
地域性品牌（仅1-2个城市有）     → 必须脱敏
```

## 案例数据展示规范

### ✅ 安全展示方式

```text
城市·品类（如"广州·奶茶店"）    ✅ — 不可追溯到个人
亏损金额（聚合/区间）            ✅ — 不展示具体个人
死因描述（通用化）               ✅ — "租金占比过高"而非"xx店被xx坑了"
统计数字（均损/品类数/案例数）   ✅ — 聚合数据无法识别个人
```

### 脱敏声明放置规则

首页的案例面板、每个付费报告页底部，都必须有脱敏声明。位置：

| 页面 | 声明位置 | 文案 |
|------|---------|------|
| 首页案例弹出面板 | 面板底部 | ⚠️ 所有案例已脱敏处理，仅展示品类+城市维度的统计信息 |
| assess/step4-report | 已内置在首页面板 | — |
| survey/step6-score | 页面底部（固定栏之上） | ⚠️ 所有分析数据基于脱敏处理的行业统计信息，不涉及具体个人或商户信息 |
| cost/step6-report | 页面底部（固定栏之上） | ⚠️ 所有案例数据已脱敏处理，仅展示品类+城市维度的统计信息 |

### ❌ 危险的展示方式

```
真实姓名 + 具体地址 + 亏损金额     ❌ — 侵犯个人信息权
抖音ID/微信ID/手机号              ❌ — 违法
原始视频截图/聊天记录截图          ❌ — 侵犯著作权/隐私权
负面评价具体品牌/商家              ❌ — 可能构成名誉侵权
"数据来源于抖音/小红书爬取"        ❌ — 暗示数据来源方式
```

## 数据源声明规范

### ❌ 不要写

- "数据来源于抖音/小红书爬取"
- "基于MediaCrawler采集"
- "爬取了xx平台的数据"
- "数据来源：xxx公开平台"

### ✅ 可以写

- "基于真实开店案例数据库分析"
- "数据来源于1377个真实开店失败案例"
- "AI深度分析"
- "数据持续更新中"

## 微信平台合规特别注意事项

1. **不展示任何第三方平台水印/标识**（抖音logo、小红书笔记截图等）
2. **不暗示为用户提供投资建议** — 使用"参考""决策辅助"而非"保证赚钱"
3. **竞品数据不使用负面措辞** — 用"距离/客单价/规模"等中性事实描述，不用"差/不行/垃圾"
4. **付费解锁内容中不包含违法导流**（群二维码、个人微信等）
