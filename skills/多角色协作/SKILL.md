# 多角色协作技能

## 触发场景

当用户提到以下关键词时，使用此技能：
- "产品模式"
- "产品架构"
- "系统设计"
- "多角色协作"

## 使用方法

### 1. 识别角色类型
分析用户需求，确定需要的角色（如：产品经理、架构师、开发者、测试等）

### 2. 分配任务
为每个角色定义职责和输出目标

### 3. 协调输出
整合各角色的分析结果，形成完整方案

## 示例输出结构

```
## 产品模式分析

### 角色 1：产品经理
- 用户需求分析
- 功能定义

### 角色 2：架构师
- 技术选型
- 系统设计

### 角色 3：开发者
- 实现方案
- 代码结构

### 总结
综合建议
```

## 注意事项

- 保持各角色视角独立
- 输出要有实操性
- 避免角色职责重叠

## 常见问题

### Skill 不触发

**历史问题**（2026-05-12 TT 环境）：该 Skill 创建后未自动触发，原因为：

1. **Skills 目录不存在** — 旧 Hermes 安装时 `skills/` 目录尚未创建。修复：确保 Hermes home 的 `skills/` 目录存在且 `SKILL.md` 文件正确放置。
2. **技能快照未加载到活跃会话** — 创建或迁移 Skill 后需要 **新开会话**（`/new` 或 New Chat）才能加载。当前会话不会自动刷新技能列表。

**诊断流程：**
```
技能列表为空？ → 检查 skills/ 目录是否有 SKILL.md
技能存在但不触发？ → 重启会话（/new），确保技能快照重新加载
```
