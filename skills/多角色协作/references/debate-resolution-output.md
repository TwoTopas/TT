# 多角色深度讨论输出格式

经会话验证（2026-06-24）的输出格式。每个子角色在输出中必须包含「冲突与裁决」表。

## 最终交付格式

### 1. 各角色核心产出摘要表

| 角色 | 核心产出 |
|------|---------|
| 🎨 UI/UX设计师 | 精确到色值/字号/间距的布局方案 + WXML伪代码 |
| 📊 产品经理 | 评审意见 + 转化策略 + 数据脱敏方案 |
| 💻 全栈开发 | 技术可行性评估 + 具体WXML/WXSS/JS实现思路 |

### 2. 冲突表（必选）

| 议题 | 设计师 | PM | 开发 | 裁决 |
|------|--------|-----|------|------|
| 具体争议点 | 方案A | 方案B | 方案C | **最终方案** |
| ... | | | | |

裁决必须明确具体，不能「等讨论」。如果三个人冲突，主持人决定。

### 3. 优先级矩阵

| 优先级 | 任务 | 说明 |
|:------:|------|------|
| 🔴 P0 | 改动项 | 不修不能上线 |
| 🟡 P1 | 改动项 | 重要但可稍后 |
| 🟢 P2 | 改动项 | 锦上添花 |
| 🔵 P3 | 改动项 | 未来版本 |

每项对应一个具体文件路径，不是模糊的「优化体验」。
