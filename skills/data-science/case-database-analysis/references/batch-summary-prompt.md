# 案例库批量汇总提示词模板

> 将多篇已生成的单案例分析报告自动汇总为带分类统计、风险排行、共性规律的标准化案例库总览。

## 输入格式

将单案例分析报告全文复制在提示词下方。若后续新增案例，在开头注明「增量更新」。

## 提示词核心结构

```
【角色设定】
资深创业风控研究专家、商业案例库架构师。
对输入的多篇创业失败单案例分析报告进行全量数据提取、分类统计、规律提炼与体系化汇总。

核心约束：
1. 所有统计数据必须来自输入的单案例报告，不得凭空编造
2. 数据不足时标注「数据不足」
3. 输出格式严格遵循以下六大模块

【强制输出六大模块】

### 一、案例库总览数据看板
4组高亮数据卡片：
1. 规模总量类：总案例数、可统计总投入、可统计总亏损
2. 亏损效率类：平均单案例亏损额、平均亏损占投入比例
3. 存活周期类：平均经营存活周期、最短/最长存活周期
4. 风险分布类：最高频失败原因TOP3、最高亏损额风险类型

### 二、全案例信息索引总表
每行对应1个案例的标准检索表。
列项：案例编号、案例名称、所属行业、初始总投入（万元）、最终亏损额（万元）、
      亏损占比、经营存活周期、核心失败主因、风险等级

### 三、多维度分类统计分析
3个维度，每个配统计表+解读：
1. 行业分布统计
2. 亏损规模分层统计（<10万/10-30万/30-50万/50-100万/>100万）
3. 失败原因归因统计（6大类）

### 四、全库共性风险矩阵与TOP排行
1. 风险层级矩阵（致命/高危/中危/低危大四象限）
2. 创业失败致命风险TOP10排行

### 五、分行业避坑方法论汇总
按行业分类提炼可复用的TOP3核心避坑规则，每条标注对应案例编号。

### 六、案例库迭代与使用说明
检索方式、更新规则、3种使用场景
```

## 批次合并策略

当全量案例超过DeepSeek 128K上下文限制时：

1. 将案例集按约50-60条/批拆分
2. 每批单独运行汇总提示词
3. 最后用合并提示词将多批汇总合并为统一总览

合并提示词要点：
```
System: 将两份批次汇总报告合并为统一总览
1. 合并统计口径：汇总计算
2. 全案例索引表：合并为完整表
3. 多维度统计重新计算
4. 风险TOP10合并
5. 分行业避坑规则合并
```
