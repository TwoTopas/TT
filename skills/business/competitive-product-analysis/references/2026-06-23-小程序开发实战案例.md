## 关键教训

### 1. WXML style属性不能混写mustache
微信小程序WXML编译器不接受 `style="color: {{var}}"` 这种混合mustache。

正确的写法：
```wxml
<!-- ✅ 纯静态 -->
<view style="color: #ff3b30; font-size: 36rpx"></view>
<!-- ✅ 纯mustache绑定（整条style都是） -->
<view style="{{item.cardStyle}}"></view>
<!-- ❌ 混写 -->
<view style="width: {{item.percent}}%"></view>
```

class可以放心用三元：
```wxml
<view class="tag {{selected ? 'act' : ''}}">...</view>
```

预计算模式（推荐）：
```javascript
// JS中预计算所有动态值
onLoad() {
  const items = raw.map(x => ({
    ...x,
    cardStyle: 'width: 30%; padding: 20rpx; border: 2rpx solid #e5e5ea;'
  }));
  this.setData({ items });
}
```

### 2. Claude Code CLI集成
`--print`模式输出到stdout，不写文件。
```bash
# 前台运行，重定向输出
/d/nodejs-v22/claude --print -p "prompt" > output.txt 2>/dev/null
# 提取代码块写入文件
python -c "import re; blocks=re.findall(r'```(?:\w+)?\n(.*?)```', open('output.txt').read(), re.DOTALL); open('dest','w').write(blocks[0].strip())"
```

- tabBar图标不存在 → 先创建images/再写app.json

### 4. 多角色开发限制
- delegate_task并发 max 3
- subagent超时 600s
- subagent不继承父skills → context中显式传递
- 子agent写小程序代码质量差 → 不应通过subagent中转
