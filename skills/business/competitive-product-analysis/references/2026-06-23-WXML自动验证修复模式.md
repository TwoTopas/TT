# WXML自动验证修复模式（2026-06-23）

## 背景

WeChat小程序WXML不支持`style`属性混写mustache表达式（如`style="color:{{var}}"`），但所有AI代码生成工具（包括Claude Code和Hermes subagents）都会产出这种代码。必须运行自动修复。

## 核心检查

```python
import re

def check_wxml_style_violations(content):
    """检查WXML文件中是否有style混写mustache的违规"""
    violations = []
    for m in re.finditer(r'style="([^"]*?){{([^}]+)}}([^"]*?)"', content):
        before = m.group(1).strip()
        after = m.group(3).strip()
        # 跳过 style="{{item.style}}" 纯mustache绑定（合法）
        if not before and not after:
            continue
        violations.append(m.group(0)[:60])
    return violations
```

## 修复流程

1. 在JS中预计算完整style字符串
2. WXML使用 `style="{{item.computedStyle}}"` 绑定
3. 扫描验证
