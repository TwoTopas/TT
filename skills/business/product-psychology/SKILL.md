# Product Psychology — Optimize Content & Buyer Journey

Based on **coreyhaines31/marketingskills** (32.7k⭐), **Predictably Irrational (Dan Ariely)**, **Influence (Robert Cialdini)**, and **Thinking, Fast and Slow (Daniel Kahneman)**.

## When to Use

- Building or optimizing a digital product for Gumroad / Etsy / direct sale
- User asks to "optimize based on psychology" / "应用心理学优化"
- Creating the buyer journey from listing → purchase → unboxing → first use
- Auditing an existing product for psychological conversion leaks
- **Pre-launch legal compliance check** (this user requires "法务审核" before launch)

## Core Principle: The Buyer Journey Has Psychology at Every Step

```
Listing → Purchase → Download/Unboxing → First Content → Regular Use → Share/Refer
```

Each step has specific psychological levers. Missing any = lost conversion or engagement.

## Step-by-Step Psychology Framework

### Step 1: Listing / Landing Page (Acquisition)

| Principle | Application | Example |
|-----------|-------------|---------|
| **Anchoring** | Compare to expensive alternative | "Compared to hiring a consultant at $150/hr..." |
| **Loss Aversion** | Frame what they lose, not what they gain | Setup without playbook = wasted weeks, silent churn |
| **Social Proof** | Show customer count + testimonials | "Join hundreds of operators", named testimonials with metrics |
| **Authority Bias** | Credibility signals early | "Trusted by community managers at [types]" |
| **Peak-End Rule** | End on emotional closing line | "Stop winging it. Start sleeping." |
| **Decoy Effect** | 3 tiers, dominated middle tier | $39 Standard ($10 less than $49 but missing key value) |
| **Scarcity** | Real scarcity, not fake | "First 100 buyers get lifetime access at this rate" |
| **Price-Quality Signal** | Don't underprice | $49 signals quality, $19 is entry |

**Listing structure (top to bottom):**
1. Emotional headline (pain → relief, physical sensation anchor)
2. Authority banner
3. Anchor comparison
4. Tiered pricing with decoy effect
5. Decoy includes warning label
6. Value justification block
7. Pricing comparison table
8. Transformation section
9. Risk reversal + Final CTA
10. Emotional closing line

### Step 2: Purchase → Post-Purchase (Delivery)

| Principle | Application |
|-----------|-------------|
| **Endowment Effect** | Immediately emphasize ownership: "This is YOURS now" |
| **Activation Energy** | Give ONE specific action they can do in <30 seconds |
| **Commitment Ladder** | First action should be trivially small and immediately completable |
| **Reciprocity** | Surprise bonus not promised on the listing page |

**Implementation checklist for ZIP delivery:**
- First file is a README with "Your First 5 Minutes" section
- README starts with emotional peak
- First action takes <30 seconds and is physical (not reading)
- Ownership language: "Yours", "You now have", "Your system"

### Step 3: First Product Experience (Unboxing)

| Principle | Application |
|-----------|-------------|
| **Peak-End Rule** | The FIRST thing they see IS the peak. Design it. |
| **IKEA Effect** | Let them customize/modify something immediately |
| **Goal-Gradient Effect** | Show a checklist with one item already checked |
| **Zeigarnik Effect** | Leave one small thing incomplete (creates pull to finish) |

### Step 4: Core Content (Sustained Value)

| Principle | Application |
|-----------|-------------|
| **Loss Aversion** | "If your engagement score drops, here's what you're losing" |
| **Social Proof** | "Hundreds of operators use these same templates" |
| **Curse of Knowledge** | Avoid jargon — write for a first-timer |
| **BJ Fogg Behavior Model** | Motivation + Ability + Prompt — all three needed |
| **Chunking** | Break content into 5-min consumable pieces |

### Step 5: Tools & Assessments (Interactive)

| Principle | Application |
|-----------|-------------|
| **Zeigarnik Effect** | Partial completion creates pull to finish |
| **Goal-Gradient Effect** | Show progress bar towards completion |
| **Endowment Effect** | Personalized results create ownership |
| **Reciprocity** | "You invested X minutes — here's what you get" |
| **Loss Aversion** | "Don't let these weak spots still be weak next month" |
### Step 6: Share & Refer (Virality)

Built into the product itself, not just the marketing.

| Principle | Application |
|-----------|-------------|
| **Social Currency** | Shareable outputs make the sharer look good |
| **Public** | Badges, leaderboards, score cards others can see |
| **Goal-Gradient Effect** | "You're #3 on the leaderboard — share to see if you move up" |
| **Peak-End Rule** | Share card includes the product name at the peak moment |

## CRITICAL: Authenticity / Humanization

**AI-generated text destroys trust.** Even if all the psychology principles above are applied perfectly, if the writing sounds like AI, the product will feel fake. Authenticity is a prerequisite, not a bonus.

### ⚠️ CRITICAL WORKFLOW: Learn from Real Human Writing First

**This user's explicit preference: before writing any product content, study real human writing patterns.** Do not assume you know what "natural" sounds like.

**Required pre-step:**

1. Search Reddit for actual discussions in the product's domain (e.g. community management, Discord server issues)
2. Extract 3-5 real threads and analyze the commenters' writing style
3. Compare with AI-generated equivalents to identify differences
4. Apply the patterns below to all product content

**Key distinctions observed from real Reddit posts vs AI writing:**

| Dimension | AI Writing | Real Human Writing |
|-----------|-----------|-------------------|
| Openings | "This addresses the challenge of..." | "Hi! I have a little problem here." / "I need some advice." |
| Admitting ignorance | Never | "I have no real idea what I'm doing wrong" / "My frustration was hard to place" |
| Numbers | "thousands of members" (vague) | "30+ new members this week" (specific) |
| Sentence rhythm | Uniform 15-25 words | Mixed: 3-word fragments + long sentences |
| Emotion | "This may be concerning" | "This drives me nuts" / "It feels like every time I join..." |
| Contractions | Inconsistent | Natural: don't, I've, didn't, that's, it's |
| Transitions | "Moreover, Furthermore, Additionally" | "So...", "Look...", "Honestly...", "The thing is..." |
| Self-correction | Never | "Wait, actually I take that back..." |
| Perfection | Perfect grammar | Fragments, occasional tangents, half-formed thoughts |

### The 10 Rules for Human-Sounding English (Reddit-Derived)

1. **Use dialog openings.** "So here's the thing." / "I need to be honest with you." — not "This guide will help you..."
2. **Admit what you don't know.** "Honestly, I'm still figuring this out too." — AI never admits ignorance.
3. **Use specific numbers, not vague ones.** "30+ members this week" beats "steady growth." "About 200 members" beats "approximately 200 members."
4. **Ask the reader questions.** "Sound familiar?" / "Anyone else dealt with this?" — AI doesn't ask because it assumes it knows.
5. **Vary sentence length dramatically.** Some 3-word sentences. Some 30-word sentences. Mix. Never uniform.
6. **Show emotion. Don't stay neutral.** "Honestly? This part drives me crazy." beats "This aspect may cause frustration."
7. **Use contractions naturally.** Don't write "you are" when "you're" is natural. Don't write "I will" when "I'll" fits.
8. **Use conversational transitions.** "So...", "Look, here's the deal.", "The thing is...", "Here's what nobody tells you."
9. **Self-interrupt and correct.** "Wait, actually that's not quite right." — AI never corrects itself mid-sentence.
10. **Don't be too correct.** Perfect grammar and uniform paragraphs = AI. Fragments, one-sentence paragraphs, and mixed punctuation = human.

### Why It Matters Psychologically
- **AI text triggers skepticism** — readers unconsciously detect uniform sentence length, em dash overuse, and synonym cycling
- **Human text triggers trust** — specific details, uneven rhythm, and personality signal a real person wrote it
- **The Psychology Paradox**: Perfectly optimized copy that reads like AI will convert WORSE than imperfect copy that reads like a human

### Integration with This Skill
After applying all behavioral psychology principles to product content, ALWAYS run a humanization pass using `writing-quality` skill:

```
Psychology optimization → Humanization (writing-quality skill) → Final product
```

### What Humanization Fixes
- Em dash overuse (#1 AI tell — replace ALL with periods/colons/commas)
- AI vocabulary (delve, seamless, leverage, robust, actionable, etc.)
- Sentence length uniformity (mix short 3-8 word sentences with long 20+ word ones)
- Rule of three overuse ("X, Y, and Z" — limit to one occurrence per piece)
- Synonym cycling (AI's fear of repetition — humans repeat the best word)
- Vague attributions ("Experts believe", "Studies show" — name sources or cut)
- Generic conclusions ("The future looks bright" — say something specific or cut)

### Step 7: ⚖️ Legal Compliance (Mandatory Before Launch)

**This user requires "法务审核" (legal audit) before any digital product launches.** Apply the following checks after the product content is finalized:

#### 7a. FTC Testimonial Compliance

**Background:** US FTC "Consumer Reviews and Testimonials Rule" (effective Oct 2024) bans fake testimonials with civil penalties.

**Checklist:**
- [ ] ALL testimonials in the product are from real users with consent
- [ ] If fabricated, add footnote: `*\* Names and identifying details have been changed. Testimonials are illustrative composites based on real outcomes.*`
- [ ] Footnote must appear on EVERY page/section where testimonials appear (listing, README, quick-start guide)
- [ ] Testimonial claims with specific numbers (member counts, revenue) are especially scrutinized

#### 7b. Trademark Compliance

**Background:** Using third-party trademarks in product names can imply endorsement or affiliation.

**Checklist:**
- [ ] Product title does NOT start with a trademarked platform name
- [ ] Format: `The [Product Name] (for [Platform] Servers)` not `The [Platform] [Product Name]`
- [ ] Add trademark disclaimer on every page that mentions the platform:
  > *"[Platform] is a trademark of [Company]. This product is not affiliated with, endorsed by, or sponsored by [Company]."*
- [ ] Platform names are used descriptively ("works with Discord") not as branding
- [ ] No platform logos used without permission
- [ ] Check brand guidelines for each mentioned platform (Discord, Circle, Slack, etc.)

#### 7c. Liability Disclaimer

**Checklist:**
- [ ] Add to listing page: disclaimer about educational purpose, no guaranteed results
- [ ] Add to ZIP README: same disclaimer
- [ ] Add to quick-start guide: same disclaimer
- [ ] Add to playbook appendices: same disclaimer
- [ ] Standard disclaimer text:
  > *"This product provides educational information and templates. Results are not guaranteed. Every [context] is different — your results will depend on your specific situation, effort, and execution. Use at your own risk. This is not legal or financial advice."*

#### 7d. Refund Policy

**This user raised a fundamental point: digital downloads cannot be physically returned. Once downloaded, the buyer keeps the files even after a refund.** Therefore:

**Two valid approaches:**
| Approach | When to use | 
|----------|-------------|
| **No Refunds** (recommended for single-purchase digital downloads) | Buyer downloads → keeps files → refund makes no logical sense. State: `Due to the digital nature of this product, no refunds will be issued once files have been downloaded.` |
| **30-Day Refund** (trust-building for new products) | Use only when building initial trust is more important than logical consistency. Expect 3-5% refund rate. |

**US law:** No federal requirement to offer refunds for digital products. "No refunds" is fully legal.
**EU law:** 14-day withdrawal right can be waived if consumer consents to immediate delivery. Gumroad as MoR handles the EU compliance checkbox automatically.

**Checklist:**
- [ ] Refund policy clearly stated on listing page
- [ ] Policy matches EULA / LICENSE.txt Section 8
- [ ] Gumroad product settings match the stated policy (set in product dashboard)
- [ ] If "No Refunds": add justification (`Due to the digital nature...`) so buyers understand

#### 7e. Commercial License Terms (if applicable)

**Checklist:**
- [ ] If offering commercial/Agency tier, define explicit terms:
  - ✅ What licensees CAN do
  - ❌ What licensees CANNOT do
  - Scope: per-project vs unlimited, with attribution vs white-label

#### 7f. Factual Claims

**Checklist:**
- [ ] All numerical claims verifiable ("Join 500+" → "hundreds of" unless provable)
- [ ] No unsubstantiated superlatives ("the best", "#1")
- [ ] No false scarcity ("limited time" that isn't real)

#### 7g. Platform Compliance

**Checklist:**
- [ ] Product type is NOT on platform's prohibited list
- [ ] [Gumroad prohibited items](https://gumroad.com/prohibited): templates, playbooks, educational content = allowed
- [ ] Not reselling PLR (private label rights) content
- [ ] Not OEM or copyrighted media

## Universal Psychology Check

| Question | Look for |
|----------|----------|
| Does this create ownership before purchase? | Free sample, free assessment, trial |
| Does the first action take <30 seconds? | Activation energy reduced |
| Is there a visible progress mechanism? | Checklists, progress bars, completion tracking |
| Does the ending echo the beginning? | Peak-End Rule |
| Is there social proof at the decision moment? | Testimonials, counts, logos near CTA |
| Does the decoy actually get dominated? | Decoy price close to target but much less value |
| Is every "get" framed as a potential "loss"? | Loss aversion |
| Does the product reinforce its own promise? | If listing promises relief, product must contain relief content |
| ✅ Legal compliance checked? | FTC, trademark, disclaimer, refund, claims, platform |

## Product Content Psychology Playbook

### Opening
```
❌ "This guide will help you get organized..."
✅ "You just took the first step toward sleeping through the night."
```
Start with emotional outcome. Use "you". Include physical sensation if possible.

### First Action Instruction
```
❌ "Read through section 1 to understand the templates..."
✅ "Find 01-launch-checklist.csv. Move the first task to 'In Progress'. Done? You've started."
```
First action = DOING, not reading. Takes 30 seconds max. Validate after completion.

### Closing
```
❌ "Your community is waiting. Go."
✅ "Your community is waiting. And now, so is your peace of mind. Stop winging it. Start sleeping."
```
Echo the emotional promise from opening. End on feeling. Make it shareable.

### Tier / Pricing Writing
```
❌ "Standard: $39 — includes all 10 templates and full guide"
✅ "Standard: $39 — full template set. But at just $10 more, Complete adds the 80-page playbook, Health Score tool, and Burnout Prevention Guide. Most operators skip this tier."
```
Decoy tier gets warning label. Target described as what you'd MISS. End decoy with social proof.
