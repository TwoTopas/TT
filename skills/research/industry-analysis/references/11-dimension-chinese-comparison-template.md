# 多维度行业对比模板（中文11维度）

当用户要求用中文对比多个行业时，使用这个11维度对比表格式。

## 表头

```markdown
| # | 行业 | 市场规模 | 利润率 | 周期性收入 | 技术缺口 | PE活跃度 | 进度 | 竞品情况 | 用户接受度 | 需求紧迫度 | 用户付费意愿 |
|---|------|---------|--------|-----------|---------|---------|------|---------|-----------|-----------|------------|
```

## 维度说明

| 维度 | 说明 | 数据来源 |
|------|------|---------|
| 市场规模 | 美元计价的市场总规模 | IBISWorld, PitchBook, 行业报告 |
| 利润率 | 毛利率区间 | 行业报告, Franchise Disclosure Docs |
| 周期性收入 | 收入是否可预测/可重复（⭐⭐⭐高/⭐⭐中/⭐低） | 服务频率+合同类型 |
| 技术缺口 | 数字化工具渗透率（极高/高/中/低/趋零） | Reddit讨论, 竞品调研 |
| PE活跃度 | 私募股权整合力度（极端/很高/中等/增长/低/无） | PitchBook, PE deal news |
| 进度 | 当前调研进展（已推进/有方案/仅调研/新发现） | 内部状态 |
| 竞品情况 | 现有竞争对手和价格 | DDG搜索, Capterra, G2 |
| 用户接受度 | 用户对软件/工具的接受程度（高/中/低） | Reddit社区情绪 |
| 需求紧迫度 | 用户有多迫切需要一个解决方案 | 社区痛点强度 |
| 用户付费意愿 | 用户愿意为工具付多少钱 | 竞品定价+社区讨论 |

## 填充顺序

1. 先填已知数据（市场/利润率/周期性）
2. 再填搜索可得数据（竞品/PE）
3. 最后填需要推理的数据（用户接受度/紧迫度/付费意愿）
4. 标注未调研的维度为"缺数据"

## 进度标记

| 标记 | 含义 |
|------|------|
| ✅ 已推进 | 有代码/产品进展 |
| 📝 有方案 | 有具体方案但未执行 |
| 📄 仅调研 | 只做了桌面调研 |
| 🆕 新发现 | 刚发现的新机会 |
| ❌ 已暂停 | 已验证不成立 |

## 示例行

```markdown
| 1 | 🐛 Pest Control | $250-300亿 | 45-55% | ⭐⭐⭐ 极高 | 中低 | 🔥极端 | 📝 有方案 | PestPac $125-200, GorillaDesk $49 | 🟡 中 | 🟡 中-高 | $50-100/月 |
```

## 输出规则

- 第一行必须是空白表头行（# / 行业 / 市场规模 / ...）
- 每行一个行业
- 最后加一个"有数据支撑的重点推荐"段落
- 标注哪些维度是缺数据的
