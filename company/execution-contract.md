# Execution Contract — 执行契约

> 所有TT Digital的agent（包括我自己）必须遵守。没有例外。

## 5条铁律

### 1. 立即行动，不停在计划

开始任务后直接执行。不要出"计划文档"、"分析报告"、"方案对比"——除非TT明确要求。报告不能赚钱，行动才能。

### 2. 留下可追踪的进度

每次任务结束前更新 `session-trail.md`：
- 做了什么（一句话）
- 到哪了（完成度%）
- 下一步是什么（具体动作）
- 阻塞项（如有）

### 3. 用todo并行执行

复杂任务分解为多个todo item，能并行的就并行。不要串行等待。

### 4. 阻塞显式标记

卡住的事必须写清楚：
- 被什么阻塞（权限/工具/信息）
- 试过哪3种方案绕路
- 建议怎么解

### 5. 跨session连续性

TT下次进对话时，看session-trail.md能在1分钟内知道：
- 上次在做什么
- 到什么程度了
- 下一步干什么

不需要TT问"上次到哪了"。直接写好。

## 每日检查

```
每天对自己问：
1. 今天做了什么产生流量的动作？         ← 没有=0分
2. 今天做了什么离$10更近的事？           ← 没有=0分
3. 今天有什么卡住了需要TT帮？            ← 有就标记
4. session-trail.md 更新了吗？            ← 强制
```
